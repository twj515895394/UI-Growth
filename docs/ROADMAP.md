# UI Growth Roadmap

## v0.1.0：基础闭环

状态：已搭建

- [x] 主 `SKILL.md`；
- [x] UI 问题分类；
- [x] 口语与专业术语映射；
- [x] 专业澄清协议；
- [x] 修改范围和风险分级；
- [x] 项目 UI 审计流程；
- [x] 设计规范治理流程；
- [x] Agent 执行合同；
- [x] 视觉、状态和响应式验收；
- [x] 项目规范模板；
- [x] 页面规范、决策日志和审计模板；
- [x] 参考项目借鉴边界。

## v0.2.0：真实项目验证

目标：在多个前端技术栈中验证 Skill 的可用性。

- [ ] Vue + Element Plus 项目案例；
- [ ] React + Ant Design 项目案例；
- [ ] 小程序 + Vant Weapp 项目案例；
- [ ] 纯 CSS/SCSS 项目案例；
- [ ] 收集常见误判并更新术语映射；
- [ ] 验证 S1–S5 范围分级；
- [ ] 优化用户确认问题的数量和质量；
- [ ] 增加多截图编号标注约定。

## v0.3.0：可选自动化工具

目标：减少项目审计和视觉验收的重复工作。

- [ ] Token/CSS Variable 扫描脚本；
- [ ] 常见尺寸、圆角、间距分布报告；
- [ ] 公共组件与局部覆盖扫描；
- [ ] Playwright 多视口截图脚本；
- [ ] 修改前后视觉差异报告；
- [ ] 控制台错误采集；
- [ ] 基线截图元数据格式。

这些工具必须是可选的，核心 Skill 不能依赖某个固定运行时。

## v0.4.0：技术栈适配参考

目标：增加实现层面的针对性，但保持核心流程无框架绑定。

- [ ] Vue/Element Plus；
- [ ] React/Ant Design；
- [ ] Tailwind CSS；
- [ ] CSS Modules；
- [ ] 小程序；
- [ ] 常见主题和 Token 系统。

适配文件只说明如何读取和修改对应栈，不改变产品方向和设计判断。

## v0.5.0：学习反馈增强

目标：让用户在真实任务中逐步形成 UI 语言。

- [ ] 场景化术语卡片；
- [ ] 常见概念对比：gap/padding、line-height/row-gap 等；
- [ ] 用户确认后的决策回顾；
- [ ] 防止重复解释用户已掌握概念的策略；
- [ ] 项目专属 UI 术语表。

## v1.0.0：稳定版本

完成条件：

- 至少覆盖三类真实技术栈；
- 多轮 UI 修改后规范不会失控；
- 用户模糊表达能稳定转换为执行合同；
- 中高风险问题能给出高质量专业选项；
- 公共组件修改有可靠回归流程；
- Skill 在 Codex、Claude Code、Cursor 等环境中经过验证；
- 文档和模板经过真实项目迭代。

## 暂不计划

当前不优先建设：

- UI Skill 市场；
- Web 展示站；
- 大型 CLI Registry；
- 固定审美风格生成器；
- 替代 Figma 或专业设计师的完整设计工具；
- 与单一前端框架深度绑定的实现。
