# UI 问题分类体系

本文件用于把用户的口语描述映射为可分析、可定位、可验收的 UI 问题。一次任务可以有多个问题，但必须区分主问题和次问题。

## 1. 布局结构

关注元素如何组织，而不是单个元素长什么样。

常见症状：

- 页面像“拼起来的”；
- 区块顺序不自然；
- 左右区域比例失衡；
- 某个区域挤压另一区域；
- 卡片列数不合适；
- 页面内容过宽或过窄。

重点检查：

- `display`、Flex、Grid；
- `grid-template-columns`；
- 固定宽度、最小宽度、最大宽度；
- 容器嵌套；
- 页面主栏与侧栏比例；
- 布局断点。

## 2. 对齐

常见症状：

- “不齐”；
- 按钮和输入框上下错位；
- 标题和右侧操作不在一条线上；
- 图标与文字看起来偏上或偏下。

区分：

- 水平对齐；
- 垂直对齐；
- 基线对齐；
- 网格边缘对齐；
- 文本与图标视觉中心对齐。

重点检查：

- `align-items`、`justify-content`；
- `line-height`；
- 控件高度；
- 图标 `viewBox` 和尺寸；
- 不一致的 Padding。

## 3. 外部间距

元素与元素、组件与组件之间的距离。

常见症状：

- “两个卡片之间太空”；
- “这个区域离下面太远”；
- “页面左右太堵/太空”；
- “列表项之间太松”。

重点检查：

- `gap`、`row-gap`、`column-gap`；
- `margin`；
- 父容器 Padding；
- Grid/Flex 分布方式；
- 空占位元素。

## 4. 内部间距

组件内容与组件边界之间的距离。

常见症状：

- 卡片“太厚”；
- 按钮显得臃肿；
- 输入框内部太空；
- 弹窗内容离边缘太远。

重点检查：

- `padding`；
- 控件高度；
- 图标与文字内部 `gap`；
- 伪元素；
- 组件库 Size Variant。

## 5. 尺寸与比例

常见症状：

- 按钮太大；
- 卡片太高；
- 头像、图标和文字比例不协调；
- 弹窗太宽；
- 操作列太宽。

重点检查：

- `width`、`height`、`min-*`、`max-*`；
- `aspect-ratio`；
- 固定高度造成的空白；
- 内容驱动尺寸；
- 图标与文字比例。

## 6. 字体与行高

用户所说的“行间距”可能不是文字行高，必须先判断对象。

常见症状：

- 多行文字太松或太挤；
- 标题和正文区别不明显；
- 表格文字显得偏上；
- 中文段落可读性差。

重点检查：

- `font-size`；
- `line-height`；
- `font-weight`；
- 字体回退；
- 文本容器高度；
- 表格单元格 Padding。

## 7. 视觉层级

常见症状：

- “没有重点”；
- 所有内容看起来一样重要；
- 次要信息抢视觉；
- 页面很乱但布局并没有明显错误。

重点检查：

- 标题、正文、辅助文字的字号和字重；
- 颜色对比；
- 主次按钮；
- 区块标题与内容距离；
- 视觉焦点数量。

## 8. 色彩与对比度

常见症状：

- 颜色太多；
- 状态颜色混乱；
- 辅助文字太深；
- 按钮不够突出；
- 暗色模式看不清。

重点检查：

- 语义颜色是否复用；
- 背景与文字对比度；
- Hover/Disabled 状态；
- 颜色是否承担了唯一的信息表达；
- 明暗主题 Token。

## 9. 边框、圆角、阴影和视觉重量

常见症状：

- “看起来很重”；
- 卡片边界太强；
- 阴影发脏；
- 每个组件都像浮起来；
- 圆角大小不统一。

重点检查：

- 边框与阴影是否重复表达层级；
- 阴影模糊、扩散和透明度；
- 圆角 Token；
- 层级是否与交互意义一致。

## 10. 信息密度

常见症状：

- 太挤；
- 太散；
- 一屏显示内容太少；
- 表格可读但操作效率低；
- 后台页面像展示型官网。

重点检查：

- 行高、单元格 Padding；
- 区块间距；
- 控件高度；
- 冗余说明；
- 展开/收起策略；
- 用户任务频率。

密度不是越高越好，应结合任务类型、阅读负担和操作频率。

## 11. 组件一致性

常见症状：

- 同样的按钮在不同页面不一样；
- 输入框高度有多种；
- 相同状态标签颜色不同；
- 弹窗标题区风格不统一。

重点检查：

- 是否绕过公共组件；
- Size/Variant 是否混乱；
- 局部样式覆盖；
- 重复实现；
- Token 是否被硬编码值替代。

## 12. 内容溢出与截断

常见症状：

- 文字超出卡片；
- 按钮被挤出；
- 表格横向滚动异常；
- 长英文、编号或 URL 撑破布局。

重点检查：

- `overflow`；
- `white-space`；
- `text-overflow`；
- `word-break`；
- Flex 子元素的 `min-width: 0`；
- Grid 最小内容尺寸。

## 13. 滚动区域

常见症状：

- 页面出现双滚动条；
- 弹窗内部无法滚动；
- 固定区域遮挡底部内容；
- 滚动后标题或按钮消失。

重点检查：

- 页面级与容器级 Overflow；
- `vh` 与 `dvh`；
- 固定头部/底部；
- Sticky 边界；
- 滚动锁定。

## 14. 浮层、弹窗和层级

常见症状：

- 下拉框被弹窗遮住；
- Tooltip 跑到错误位置；
- 遮罩层层级异常；
- 弹窗超出屏幕。

重点检查：

- Portal/Teleport 挂载位置；
- `z-index` 体系；
- 定位上下文；
- 视口边界碰撞；
- Focus Trap。

## 15. 状态完整性

至少考虑：

- Default；
- Hover；
- Active；
- Focus；
- Disabled；
- Loading；
- Empty；
- Error；
- Success；
- Readonly；
- Skeleton。

用户只指出默认态问题时，也要判断修改是否会破坏其他状态。

## 16. 交互反馈

常见症状：

- 点击后没反应；
- 不知道是否保存成功；
- 重复点击；
- 删除操作过于突然；
- 校验错误离输入位置太远。

重点检查：

- 反馈位置；
- 状态持续时间；
- 重复提交；
- 乐观更新与回滚；
- 焦点恢复；
- 危险操作确认。

## 17. 响应式与断点

常见症状：

- 小屏溢出；
- 按钮随机换行；
- 表单列数不合理；
- 内容顺序在移动端不合适；
- 仅在某个宽度下错位。

重点检查：

- 断点来源；
- Grid/Flex 换行；
- 表格降级策略；
- 固定宽度；
- 触控区域；
- 内容优先级。

## 18. 可访问性

重点检查：

- 键盘可操作；
- Focus 可见；
- 图标按钮可访问名称；
- 表单 Label 和错误关联；
- 对比度；
- 动画减弱偏好；
- 语义结构。

## 19. 性能导致的视觉问题

常见症状：

- 页面加载时跳动；
- 动画卡顿；
- 滚动掉帧；
- 图片加载后布局变化；
- 弹窗打开时闪烁。

重点检查：

- Layout Shift；
- 图片尺寸占位；
- 骨架与真实内容结构；
- 动画属性；
- 大面积 Blur；
- 重复渲染。

## 20. 设计规范偏离

当某个局部实现与项目已确认规则冲突时，标记为规范偏离，而不是单纯“审美不好”。

报告必须说明：

- 偏离的规范；
- 当前实现；
- 是修复偏离，还是建立合理例外；
- 是否需要更新决策日志。
