# UI 视觉与响应式验收

UI 修改必须经过实际渲染验证。编译、Lint 和单元测试通过只能证明代码层面没有明显错误，不能证明视觉任务完成。

## 一、验收层级

根据修改范围选择：

### V1 局部验收

适用于单个元素和低风险样式调整。

检查：

- 目标元素；
- 相邻元素；
- 默认状态；
- 当前视口；
- 控制台错误。

### V2 页面验收

适用于组件和页面级修改。

增加检查：

- 页面主要区域；
- 窄桌面；
- 相关交互状态；
- 文本溢出；
- 页面滚动。

### V3 系统回归

适用于公共组件或全局 Token 修改。

增加检查：

- 多个代表页面；
- 不同组件 Variant；
- 明暗主题；
- 移动端；
- 可访问性；
- 截图基线差异。

## 二、修改前基线

尽可能记录：

- 页面路由；
- 数据状态；
- 视口尺寸；
- 浏览器；
- 当前截图；
- 目标区域放大图；
- 关键 Computed Style；
- 已存在控制台错误。

修改前后必须尽量使用相同数据、视口和滚动位置。

## 三、截图对比

对比时不要只看“更好看了吗”，至少检查：

- 用户指出的问题是否消失；
- 是否出现新的错位；
- 内容是否被压缩或截断；
- 视觉层级是否改变；
- 相邻区域是否被牵连；
- 页面整体密度是否与项目一致；
- 改动是否超出用户目标。

截图无法证明 Hover、Focus、Loading 等动态状态，必须单独触发验证。

## 四、视口选择

优先使用项目文档、断点配置和真实目标设备。

项目未说明时，可将以下作为建议值而非硬性标准：

- 桌面：`1440 × 900`；
- 窄桌面：`1280 × 800`；
- 小型笔记本：`1024 × 768`；
- 移动端：`375 × 812`。

若项目明确只支持企业桌面端，不必为了形式增加移动端改造，但至少检查项目声明的最小桌面宽度。

## 五、响应式检查表

- 是否出现意外横向滚动；
- Flex/Grid 是否在正确断点换行；
- 按钮组是否被拆散；
- 表单 Label 与控件是否仍可读；
- 表格是否使用既定滚动、折叠或卡片策略；
- 弹窗是否超出视口；
- 固定底部操作是否遮挡内容；
- 文本是否截断；
- 图标是否缩小到不可识别；
- 触控区域是否过小；
- 信息顺序是否符合优先级；
- 键盘弹出时关键操作是否仍可用。

## 六、状态检查表

根据任务相关性选择：

### 输入和表单

- Focus 边界可见；
- Error 信息靠近字段；
- Disabled 与 Readonly 可区分；
- 长文本不撑破；
- 校验出现后布局不异常跳动。

### 按钮

- Hover、Active、Focus；
- Disabled；
- Loading 时宽度稳定；
- 图标和文字对齐；
- 重复点击被控制。

### 列表和表格

- Loading；
- Empty；
- Error；
- 长内容；
- 大数据量；
- 固定列；
- 分页；
- 横向滚动。

### 弹窗和抽屉

- 打开和关闭；
- 内容过长；
- 键盘焦点；
- Footer；
- 危险操作确认；
- 小视口；
- 下拉、日期选择等浮层。

## 七、设计规范一致性

检查：

- 是否复用项目 Token；
- 是否引入新的魔法值；
- 是否绕过公共组件；
- 是否改变默认 Variant；
- 是否与同类页面间距、圆角和控件高度一致；
- 是否需要记录例外；
- 是否需要更新设计决策日志。

## 八、视觉回归基线

重要页面可保存：

```text
docs/ui/baselines/
├── <page>-desktop.png
├── <page>-narrow-desktop.png
└── <page>-mobile.png
```

基线截图需要记录：

- 页面路由；
- 视口；
- 数据状态；
- 日期或版本；
- 主题；
- 是否存在动态内容。

不要把含随机数据、时间、动画中间帧的截图直接当作稳定基线。

## 九、失败条件

出现以下任一情况，不应报告完成：

- 用户指出的问题仍明显存在；
- 只在一个视口正确；
- 出现新增横向滚动；
- 内容被遮挡或截断；
- 相关状态没有验证；
- 修改公共组件却未抽样其他页面；
- 控制台出现新增错误；
- 修改效果依赖偶然数据；
- 通过大量 `!important` 或绝对定位勉强实现；
- 没有实际渲染证据却声称视觉验收通过。

## 十、无法运行项目时

如果缺少依赖、环境、账号或数据：

1. 完成静态代码检查；
2. 明确哪些结论来自代码；
3. 不声称截图或响应式已经通过；
4. 提供可执行的验收步骤；
5. 将状态标为“待运行验证”。
