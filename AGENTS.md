# AGENTS.md

本仓库维护 `UI Growth` Skill。所有人类贡献者和代码 Agent 必须遵守以下规则。

## 1. 产品方向

UI Growth 的核心用户是缺少 UI/CSS/设计理论基础，但需要借助代码 Agent 完成前端开发的用户，尤其是后端开发者。

不要把项目改造成：

- 通用 UI Skill 聚合市场；
- 某个框架或组件库专属 Skill；
- 自动追求某一种审美风格的提示词；
- 单纯的 CSS 检查清单；
- 只生成设计文档而不处理具体 UI 修改的工具。

核心闭环始终是：

`口语/截图 → 意图理解 → 专业诊断 → 必要确认 → 执行合同 → 最小修改 → 视觉验收 → 规范沉淀 → 渐进学习`

## 2. Skill 文件

- 主入口：`skills/ui-growth/SKILL.md`；
- 深度规则放在 `skills/ui-growth/references/`；
- 可复制文档放在 `skills/ui-growth/templates/`；
- 示例放在 `examples/`；
- 项目设计和路线放在 `docs/`。

主 Skill 应保持完整但可导航。专项细节优先放入 references，并在主 Skill 中按需引用。

## 3. 内容规则

- 使用中文解释，Skill 标识符和文件名使用简洁英文；
- 区分观测事实、专业推断、建议方案和用户决策；
- 不凭截图宣称精确 CSS 数值；
- 不把单页实现直接升级成项目规范；
- 不强制 Tailwind、React、Vue 或任何组件库；
- 不把“高级感”绑定到渐变、毛玻璃、大圆角或复杂动画；
- 不鼓励大范围无关重构；
- 所有验收规则必须能够观察或验证。

## 4. 修改主 Skill 时

检查是否仍然覆盖：

- 页面视觉问题分类；
- UI 专业术语转换；
- 修改范围识别；
- 目标效果补全；
- 专业澄清与建议；
- Agent 执行合同；
- 项目规范生成和维护；
- 修改后截图验收；
- 响应式与状态检查；
- 规范一致性检查；
- 渐进式用户学习。

## 5. 模板规则

模板中的内容必须：

- 可直接复制到目标项目；
- 不预填未经证实的像素值；
- 包含状态或证据字段；
- 允许项目适配现有目录；
- 不制造与目标项目现有设计文档平行冲突的体系。

## 6. 版本与变更

- 行为规则变化需要更新版本和 `CHANGELOG.md`；
- 新增系统级能力需要更新 `docs/ARCHITECTURE.md`；
- 新增长期计划需要更新 `docs/ROADMAP.md`；
- 引入外部思想时，在 `docs/REFERENCE-REVIEW.md` 说明借鉴边界。
