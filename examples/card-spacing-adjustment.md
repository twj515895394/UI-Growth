# 示例：两个卡片之间空白太大

## 用户输入

> 这两个卡片边上怎么这么多空白，能不能紧凑一点？其他地方不要乱动。

用户同时提供一张页面截图，并在两个并排卡片之间画框。

## UI Growth 的理解

### 用户真正意图

希望提高两个卡片区域的横向信息密度，但不希望压缩卡片内容，也不希望 Agent 对整个页面做“全面美化”。

### 问题分类

- 主问题：外部间距；
- 次问题：信息密度；
- 可能范围：S2 当前卡片容器；
- 初始风险：低到中，取决于容器是否为公共组件。

### 需要区分

空白可能来自：

1. 父容器 `column-gap`；
2. 卡片自身 `margin`；
3. 页面容器 Padding；
4. Grid 列宽或 `justify-content: space-between`；
5. 卡片内部 Padding；
6. 固定高度造成的视觉空白。

不能直接把所有 Padding 一起缩小。

## 项目检查结果（示例）

- 父容器：`display: grid`；
- 当前 `column-gap: 32px`；
- 卡片内部 `padding: 20px`；
- 项目其他两个双列页面使用 `gap: 16px`；
- 当前容器为页面局部样式，不是公共组件。

## 专业转换

> 当前问题主要是双列卡片容器的水平间距过大，导致横向信息密度偏低。卡片内部 Padding 与项目其他卡片一致，不建议一起压缩。建议将当前页面卡片容器的 `column-gap` 从 `32px` 调整为项目已有的 `16px`，只影响当前区域。

## 是否需要询问

不需要。

原因：

- 用户明确只调整两个卡片之间；
- 代码证据已经确定空白来源；
- 项目存在可复用的 16px 模式；
- 当前修改是页面局部、可回退且不影响公共组件。

## 执行合同摘要

- 页面：客户概览；
- 目标：双列统计卡片容器；
- 范围：S2；
- 修改：`column-gap: 32px → 16px`；
- 保持：卡片内部 Padding、卡片宽度比例、业务内容、接口和交互不变；
- 验收：1440px 和 1280px 下保持双列，无横向滚动，卡片内容不截断。

## 修改后报告示例

### 修改内容

- `src/pages/customer/Overview.module.scss`
- `.summaryGrid` 的 `column-gap` 从 `32px` 调整为 `16px`。

### 验收

- 1440×900：通过；
- 1280×800：通过；
- 无横向滚动；
- 卡片内容无截断；
- 相邻区域位置不变；
- 控制台无新增错误。

### 本次学到

- `gap` 控制同级卡片之间的距离；
- `padding` 控制卡片内容到卡片边界的距离；
- 本次空白发生在卡片之间，因此只改 `gap`，不改 `padding`。
